What SimpleReview can fix on Bitrix

Component templates in /local/templates/[theme]/components/bitrix/
Site template files in /local/templates/[theme]/header.php, footer.php
Language strings in /local/templates/[theme]/lang/ru/
Init hooks in /local/php_interface/init.php
CSS/JS in /local/templates/[theme]/styles.css, script.js
Removing the "Powered by 1С-Bitrix" line / editing the copyright
Cache reset on "critical error" (/bitrix/cache/, /bitrix/managed_cache/)
Public form settings (captcha, validation)

Works with the "Start" / "Small Business" / "Business" / "Web-Cluster" editions. Never edits files inside /bitrix/ (the core) — only /local/ overrides, so updates won't wipe your changes. We're shipping a simplereview module to 1С-Bitrix: Marketplace (in approval). Standard installer: Marketplace → Catalog → Search "SimpleReview" → Install. Adds an "Edit with SimpleReview" button next to every template/component in the admin.

How SimpleReview works on Bitrix sites

1

Install the Chrome extension

Auto-detects Bitrix via <meta name="generator" content="Bitrix Site Manager"> + JS signatures.

2

Connect your site's Git/SFTP

GitHub/GitLab/Bitbucket/self-hosted Gitea or direct SFTP. SimpleReview indexes /local/ on first connection.

3

Click an element on the storefront

Any block, button, footer copyright, component element. Captures DOM + Smarty/PHP source path.

4

Get a Pull Request

Type "remove the Bitrix copyright" or "fix the captcha on the form". The agent edits the right file in /local/templates/.../footer.php and opens a PR. You merge it or reject it.

FAQ

Can I customize a Bitrix site without a developer?
For 80% of tasks — yes. Templates, language strings, captcha, copyright. For 1С integration, custom payment, migration — pair with a Vibers human review.
How do I install SimpleReview on a Bitrix site?
Today: Chrome Extension + connect Git/SFTP. Soon: a native module on 1С-Bitrix: Marketplace (in approval). Standard installer: Marketplace → Catalog → Search "SimpleReview" → Install.
Which Bitrix editions are supported?
Site Management (Start / Small Business / Business / Web-Cluster). All versions 16.0+. Bitrix24 SaaS is not supported (no file access). Bitrix24 Enterprise on-premise — supported.
Is this safe for production?
SimpleReview never pushes directly to the main branch. Every change is a PR you review and merge. Use a staging clone + git workflow.
How is this better than a $30-50/hour Bitrix freelancer?
Speed and starting cost. A freelancer needs 1-3 hours just to clone the repo and locate the file. SimpleReview already knows the /local/ structure and opens a PR in seconds.
What about complex modules — payment, 1С integration, Bitrix24 ↔ on-premise migration?
That's work for a real engineer. SimpleReview can prep a draft PR, but the final merge needs a human reviewer. Vibers does exactly that — reviews PRs before merge.
