How SimpleReview works on Shop-Script

1

Install the Chrome extension

Install SimpleReview from the Chrome Web Store. Click the extension icon while your Shop-Script storefront is open.

2

Auto-detect Webasyst

Platform is detected automatically via <meta name="generator" content="Webasyst"> and the typical HTML signatures. SimpleReview switches into Webasyst mode immediately.

3

Connect Git or SFTP

GitHub, GitLab, Gitea, Bitbucket — or direct SFTP. SimpleReview indexes wa-apps/, wa-config/, themes and plugins on first connection.

4

Click → get a PR

Click the broken element, describe the fix. The agent finds the file in wa-apps/shop/themes/[theme]/ or the plugin and opens a Pull Request — with a note about php wa.php install [plugin_id] if the plugin needs reinstalling.

FAQ

Can I customize a Shop-Script site without a developer?
For most theme and content tweaks — yes. Smarty templates in wa-apps/shop/themes/[theme]/, CSS, locale files, hooks. SimpleReview opens a PR in seconds. For 1С integration, custom payment, version migrations — pair with a Vibers human review.
How do I install SimpleReview on Shop-Script (Webasyst)?
Today: install the Chrome Extension and connect a Git repo or SFTP. Coming soon: a native plugin in Webasyst Store (in approval). Once published — install via the Installer: php wa.php install simplereview.
Which Webasyst versions are supported?
Webasyst Framework 2.x + Shop-Script 9 and 10 — full support. Supported themes: default, Hypermarket, Dummy, and any custom theme that follows the override convention.
Does it work with other Webasyst apps — CRM, Site, Mailer, Blog?
Yes. Webasyst is a framework powering Shop-Script, Webasyst CRM, Site, Mailer, Blog, Helpdesk. They all use the same wa-apps/[app]/ structure, so SimpleReview works the same with themes, plugins and hooks across all of them.
Is this safe for production Shop-Script?
SimpleReview never pushes directly to the live branch. Every change is a Pull Request you review and merge. Worst case — you reject the PR, prod is untouched. For critical fixes we recommend a staging clone (still cheaper than paying $50/hour).
How is this cheaper than a $30-50/hour freelancer?
A webasyst.com freelancer typically charges $30-50/hour and burns 1-2 hours just on onboarding — cloning the repo, setting up locally, locating the right file in wa-apps/shop/themes/. The fix itself is one line. SimpleReview already knows the Webasyst file layout and opens a PR in seconds. For complex work (1С integration, payment, migration) — pair with Vibers.
