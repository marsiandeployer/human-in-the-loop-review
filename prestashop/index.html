How SimpleReview works on PrestaShop

1

Install the Chrome Extension

Add SimpleReview to Chrome. Auto-detects PrestaShop via <meta name='generator' content='PrestaShop'> and the front-office HTML signatures.

2

Connect your store's Git or SFTP

GitHub, GitLab, Gitea, Bitbucket, or direct SFTP. SimpleReview indexes themes/, modules/, and override/ on first connection.

3

Click the element on your storefront

Hover any block, button, footer credit, broken module hook output. SimpleReview captures the rendered DOM and the Smarty/Twig source path automatically.

4

Get a Pull Request, not a ticket

Type "remove this expired banner" or "fix the newsletter block on mobile". The agent edits the right themes/[theme]/templates/_partials/footer.tpl (or wherever the element lives) and opens a PR. You merge or decline.

FAQ

Can I edit my PrestaShop store without a developer?
For most cosmetic and structural edits — yes. Theme files (themes/[theme]/templates/), CSS, language strings, hook overrides. SimpleReview opens a PR for those in seconds. For payment-gateway integrations, multi-store config, and large migrations — pair with Vibers human review.
How does SimpleReview install on PrestaShop?
Today: install the Chrome Extension and connect your Git repo or give SFTP credentials. Tomorrow: native PrestaShop module on PrestaShop Addons (in submission — usual cycle 2-6 weeks). Once approved, install via Module Manager → Upload a module → activate.
Which PrestaShop versions are supported?
PrestaShop 1.7.x and 8.x — full support (Smarty front-office + Twig back-office). PrestaShop 1.6.x — limited (legacy template engine, no Twig in admin). Tested on Classic, Warehouse, Alysum themes; should work on any theme that follows the standard override pattern.
Is it safe for a live PrestaShop store?
SimpleReview never pushes to your live branch directly. Every change is a Pull Request you review and merge. Pair with a staging clone (most managed PrestaShop hosts include one) and the worst case is a PR you reject.
How is this different from hiring a freelancer for €30-50/hr?
Speed and recurring cost. A freelancer takes 1-3 hours just to clone the repo, set up an environment, and locate the file — the actual edit is often one line. SimpleReview already knows the PrestaShop file layout (themes/[theme]/templates/, modules/, override/) and opens the PR in seconds. For complex work — custom payment, migration, ERP integration — you still want a human; that's what Vibers is for.
What about paid PrestaShop Addons modules — Warehouse, Alysum, etc.?
SimpleReview reads the module's published files but won't modify a paid module's licensed source unless you explicitly opt in (you signed the modification rights). For module bug reports, escalate to the vendor; for theme-level overrides on top of a paid module — works exactly as for any theme.
