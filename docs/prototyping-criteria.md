# Prototyping Criteria — Landing Page (vibers.onout.org)

## Universal Criteria

1. **Clarity** — Can someone understand what this service is and how to start within 5 seconds of landing?
2. **Visual Identity** — Does it have a distinct, intentional look — not a generic template?
3. **Interaction Completeness** — Do all clickable/interactive elements (buttons, links, collapsibles, copy, language switch) actually work?

## Landing-Page Specific Criteria

4. **CTA Effectiveness** — Is the primary call-to-action (install App / copy skill command) clear, visible, and compelling? Does it drive action?
5. **Persona Fit** — Does the page resonate with the target audience (solo builders, vibecoder teams, Claude Code / Cursor users who need human code review)?
6. **Competitive Differentiation** — Does the page clearly explain WHY this is better than SonarQube, CodeRabbit, or hiring a senior dev?
7. **Trust & Credibility** — Are there enough trust signals (real examples, honest disclaimers, security transparency, Trustpilot, reviewer profile)?

## Scoring

Each criterion scored 1-10. Specific evidence required for every score.

**PASS:** average >= 7, no criterion below 5
**FAIL:** average < 7 or any criterion below 5
