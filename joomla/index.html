How SimpleReview works on Joomla

1

Install the Chrome Extension

Add SimpleReview to Chrome. The native Joomla module is currently in approval at the Joomla Extensions Directory (JED) — once published, install via Extensions → Manage → Install.

2

Auto-detects Joomla via generator meta

SimpleReview reads <meta name="generator" content="Joomla! - Open Source Content Management"> and the front-end signatures, then identifies the active template (Cassiopeia, Helix Ultimate, YOOtheme, Quix, etc.).

3

Connect your site's Git or SFTP

GitHub, GitLab, Gitea, Bitbucket, or direct SFTP. SimpleReview indexes templates/, language/, and modules/ on first connection.

4

Click element → get a Pull Request

Hover any block, button, "Powered by" credit, broken module output. Type the change in plain language. The agent edits the right templates/[template]/html/com_[component]/ override and opens a PR. After merge, run Joomla System → Maintenance → Clear Cache → Clear All (and Tools if needed) to flush stuck visual states.

FAQ

Can I edit my Joomla site without a developer?
For most cosmetic and structural edits — yes. Template overrides in templates/[template]/html/com_[component]/, language strings, module overrides, custom CSS/JS. SimpleReview opens a PR for those in seconds. For Joomla extension development, multilingual config, and major migrations — pair with Vibers human review.
How does SimpleReview install on Joomla?
Today: install the Chrome Extension and connect your Git repo or give SFTP credentials. Tomorrow: native Joomla module shipping to the Joomla Extensions Directory (JED) — currently in approval. Once approved, install via Extensions → Manage → Install → Upload Package File.
Which Joomla versions are supported?
Joomla 4.x and 5.x — full support (modern template structure with templates/[template]/html/ overrides). Joomla 3.10 — limited (EOL in August 2023, legacy MVC). Tested on Cassiopeia and major custom templates.
Does it work with Helix Ultimate, YOOtheme, Quix templates?
Yes — any custom Joomla template that follows the standard override path templates/[template]/html/com_[component]/ works. Helix Ultimate, YOOtheme Pro, Quix, JS_Unlimited, Astroid, and the default Cassiopeia template are all supported. SimpleReview detects the active template and writes the override into the correct theme folder, never touching components/com_[name]/.
Is it safe for a live Joomla site?
SimpleReview never pushes to your live branch directly. Every change is a Pull Request you review and merge. After deploy, run Joomla System → Maintenance → Clear Cache → Clear All (and Tools if needed) to flush stuck visual states. Worst case is a PR you reject.
How is this cheaper than hiring a freelancer for $30-50/hr?
Speed and recurring cost. A Joomla freelancer takes 1-3 hours just to clone the repo, set up an environment, and locate the override file — the actual edit is often one line. SimpleReview already knows the Joomla override system (templates/[template]/html/com_[component]/, language/[lang]/, module overrides) and opens the PR in seconds. For complex work — custom extensions, J3→J5 migration, multilingual setup — pair with Vibers human review.
